UI & Data Model ต่อทางเลือก · สำหรับ PO / ผู้บริหาร

เลือกทางเลือกไหน — ลูกค้าเห็นยังไง & ระบบเก็บข้อมูลยังไง

เอกสารพี่น้องของ "สรุปเพื่อการตัดสินใจ" (20260630_coupon_brief.html) — เล่มนี้โชว์ หน้าจอจำลอง ที่ลูกค้า/พนักงานจะเห็น (ต้องจ่ายอะไร ก่อน-หลัง) และ โครงสร้างข้อมูล ที่ระบบใช้เก็บ ของแต่ละทางเลือก เน้นทางเลือก A (แนะนำ).

ตัวอย่างที่ใช้: ค่าห้อง ฿17,000 + ค่าแขกเสริม ฿1,500 + ค่าสัตว์เลี้ยง ฿800 มัดจำ 60% ของค่าห้อง แนะนำ: ทางเลือก A
สรุป + ตัวเลขตัวอย่าง ศัพท์ (ฉบับย่อ) ทางเลือก A — แนะนำ ทางเลือก B — จ่าย 2 งวดเต็ม ทางเลือก C — เปลี่ยนแค่เลขที่แสดง เทียบ UI & Data 3 ทาง ต่อจากนี้
฿19,300ยอดเต็มของการจองห้อง 17,000 + แขกเสริม 1,500 + สัตว์ 800
฿10,200จ่ายตอนจอง (มัดจำ)60% ของค่าห้องเท่านั้น
฿9,100จ่ายหน้างาน (check-in)ค่าห้องคงเหลือ + แขกเสริม + สัตว์

อ่านสั้น ๆ ก่อน

  • หัวใจของเรื่อง: เปลี่ยน "มัดจำ" ให้คิดจาก ค่าห้องอย่างเดียว (ไม่รวมค่าสัตว์/แขกเสริม) แล้วให้จ่ายส่วนที่เหลือ หน้างานตอน check-in.
  • ลูกค้าจะเห็น 2 กลุ่มชัด ๆ บนใบสรุปการจอง: กล่องสีฟ้า "จ่ายตอนนี้ (มัดจำ)" และกล่องสีเหลือง "จ่ายหน้างาน".
  • ระบบเก็บข้อมูลยังไง: แต่ละยอดคือ "บิลย่อย 1 รายการ" (charge) ทางเลือก A แค่ ติดป้าย "เก็บหน้างาน" (collect_on_site) บนบิลที่ต้องไปเก็บหน้างาน — เพิ่มช่องเดียว ไม่แตะของเดิม.
  • ความต่างหลักของ 3 ทางเลือก อยู่ที่ "ระบบเก็บข้อมูล/เก็บเงินจริงต่างกันแค่ไหน" — A เก็บตรงกับที่โชว์, B เพิ่มสถานะ "รอเก็บเงินคงเหลือ", C โชว์เลขใหม่แต่หลังบ้านเก็บเลขเดิม (ไม่ตรง).

เทียบ UI & Data Model ทั้ง 3 ทาง
มุมมองA — มัดจำจากค่าห้อง + ป้ายหน้างานB — จ่าย 2 งวดเต็มC — เปลี่ยนแค่เลขที่แสดง
ลูกค้าจ่ายตอนจอง฿10,200 (มัดจำค่าห้อง)฿10,200 (มัดจำค่าห้อง)โชว์ ฿10,200 แต่ตัดจริง ฿11,580
ค่าสัตว์/แขกเสริมเก็บหน้างาน 100%เก็บหน้างาน 100%ยังเก็บล่วงหน้า (ปนในมัดจำ)
หน้าจอแยก "ตอนนี้/หน้างาน"มี ชัดเจนมี + สถานะค้างชำระมี (แต่ตัวเลขลวง)
เปลี่ยนโครงสร้างข้อมูล+1 ช่อง (collect_on_site)+1 ช่อง + ฟิลด์สถานะการจองไม่เปลี่ยน
ตัวเลขหน้าจอ = เงินจริงตรงตรงไม่ตรง
ขนาดงานปานกลาง (M)ใหญ่ (L)เล็ก (S)
ตอบโจทย์ธุรกิจเต็มเต็ม + ยืดหยุ่นไม่จริง (หลอกตา)
สรุป: ทางเลือก A ให้ "หน้าจอที่ตรงกับเงินจริง" ด้วยการเพิ่มข้อมูลแค่ช่องเดียว — ได้ผลเต็มโดยเสี่ยงน้อย และต่อยอดเป็น B (จ่าย 2 งวดเต็ม) ได้ภายหลังโดยไม่ต้องรื้อ. C ตัดทิ้งเพราะตัวเลขหน้าจอไม่ตรงเงินจริง.
